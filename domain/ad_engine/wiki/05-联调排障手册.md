# 联调排障手册

## 1. 适用范围

用于广告在线投放链路的快速排障，覆盖召回、两率预估、出价、精排四大环节。

## 2. 一次完整排障最小信息

- 请求时间窗口（精确到分钟）。
- trace_id / request_id。
- 广告位与终端信息。
- 计划 ID / 创意 ID。
- 实际曝光、点击、转化回传数据。

## 3. 标准排查顺序

1. 看召回是否拿到足够候选（CandidateCount）。
2. 看两率预估是否异常漂移（pCTR/pCVR 与历史对比）。
3. 看出价是否被预算或风控压制。
4. 看精排是否因为约束导致高分广告出局。

## 4. 快速判断模板

### 4.1 无量问题

- 召回候选过低：优先排查索引、过滤器、特征缺失。
- 召回正常但胜率低：优先排查出价参数和竞争环境。

### 4.2 成本问题

- 成本高但转化少：优先排查 pCVR 偏差与校准策略。
- 成本稳定但消耗慢：优先排查预算 pacing 与竞价胜率。

## 5. 日志建议字段

- `trace_id`
- `recall_sources`
- `candidate_count_before_filter`
- `candidate_count_after_filter`
- `pctr`
- `pcvr`
- `bid`
- `ecpm`
- `rank_score`
- `drop_reason`

## 6. 联调建议

- 每次变更至少提供一个“正常样本”和一个“异常样本”。
- 联调阶段开启详细日志，线上阶段保留采样日志。
- 对关键配置（目标 CPA、预算策略、探索开关）建立变更审计。
