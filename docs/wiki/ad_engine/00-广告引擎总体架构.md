# 广告引擎总体架构

## 1. 目标

广告引擎的核心目标是在每次请求中完成以下动作：

1. 在毫秒级时延内召回候选广告。
2. 估计每个候选的点击率与转化率。
3. 结合业务目标完成出价计算。
4. 通过精排模型输出最终展示序列。

## 2. 在线链路

典型在线投放链路如下：

`请求接入 -> 召回 -> 两率预估 -> 出价 -> 精排 -> 计费/日志回传`

说明：

- 召回负责“找全候选”。
- 两率预估负责“估计效果”。
- 出价负责“把效果映射为收益和竞争力”。
- 精排负责“在约束下选最优展示组合”。

## 3. 关键指标

- pCTR：预测点击率。
- pCVR：预测转化率。
- eCPM：千次展示预期收益，常用作排序核心指标。
- ROI：投入产出比。
- WinRate：竞价胜出率。
- FillRate：请求填充率。

## 4. 常见策略约束

- 预算约束：日预算、分时预算、预算平滑（Pacing）。
- 风控约束：频控、黑白名单、素材审核状态。
- 业务约束：新客探索、保量广告位、品牌安全。

## 5. 线上问题定位建议

- 召回不足：重点看候选量、过滤器命中率、索引延迟。
- 两率波动：重点看特征新鲜度、模型版本、校准参数。
- 出价异常：重点看目标 CPA/ROI 配置、预算节奏、竞价日志。
- 精排抖动：重点看特征缺失、重排约束冲突、探索权重。
