# 06-会话记忆与上下文管理子系统设计（按当前实现）

## 1. 目标

在不维护“阶段机”的前提下，保留足够的跨轮上下文，支持：

- 省略主语的追问
- 基于上一轮分析继续提问
- 代码请求复用最近分析结果

## 2. 记忆来源

`load_context` 节点从 `history` 中提取：

- 最近一条知识问答消息
- 最近一条问题分析消息（含 `analysis`）

并生成当前轮上下文：

- `module_name`
- `module_hint`
- `active_module_name`
- `active_topic_source`（`history_memory` / `current_query`）
- `last_analysis_result`
- `last_analysis_citations`

## 3. 复用规则

- 当输入是指代型追问（如“这个问题”“那块逻辑”）时，优先复用历史模块。
- 当输入像“代码在哪/哪个文件”但未明确模块时，优先复用历史模块。
- 其他情况以当前输入重新推断模块。

## 4. 当前设计边界

当前不再维护以下会话字段：

- `active_task_stage`
- `pending_action`
- `task_stage`
- `transition_type`

系统策略是“每轮重判意图 + 轻量上下文复用”。
