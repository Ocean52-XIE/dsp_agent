# 08-观测评估与运维子系统设计（按当前实现）

## 1. 观测目标

- 可追踪每轮请求经过的节点路径
- 可查看最终路由和证据使用情况
- 可统计反馈质量与告警

## 2. 当前观测入口

- 运行日志：`workflow.runtime_logging`
- 回合观测落库：`src/workflow/observability/postgres_store.py`
- API：
  - `GET /api/observability/summary`
  - `GET /api/observability/alerts`

## 3. 当前调试信息（finalize_response）

`assistant_message.debug`（开启 verbose 时）包含：

- `scene_label`
- `domain_relevance`
- `latency_ms`
- `route`
- `graph_backend`
- `graph_path`
- `verbose`（检索计划、检索画像、证据统计、node_trace）
- `llm_call_status`

## 4. 关键指标建议

- 路由分布：knowledge_qa / issue_analysis / code_generation / out_of_scope
- 各节点耗时 P50/P95
- 检索命中率与证据数量
- 用户反馈 helpful rate
- LLM 调用可用率与回退率

## 5. 兼容说明

存储层表结构中仍存在历史调试列（如 `task_stage`、`transition_type` 等）用于兼容旧数据。
当前工作流已不再维护这些字段值。
