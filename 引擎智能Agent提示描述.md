# 项目背景
我的系统中现在有20万行的代码库、100篇业务知识库wiki、100篇历史案例分析文档，我想基于langchain/langgraph框架和web界面交互来实现一个智能问答/问题分析系统

## 目标功能：
系统支持跟用户进行多轮问答：
1、根据用户输入判断是业务知识的问答还是具体问题的分析；
2、如果判断用户的输入与系统无关时则提示用户并退出此次会话；
3、如果判断用户输入的是具体问题，则根据文档和代码分析问题所在的模块和问题产生的原因以及问题修复的方案，并确认是否需要为用户提供代码实现
4、如果用户输入的是知识问答，则根据文档和代码回答用户的问题
